## What changed

### next.config.ts
- Added `output: "export"` — tells Next.js to produce static HTML files in `out/`
- Added `trailingSlash: true` — required for Capacitor file-based routing (e.g., `session/_/summary/index.html`)

- Removed `server.url: "https://gymhub-inky.vercel.app/"` — app now loads from local bundled assets

## Known issues / things to verify on real device

1. **Deep link cold launch behavior** — `server.errorPath: "index.html"` is set, but this behavior has not been tested on a physical device or simulator. The SPA fallback is the critical path for `/session/[uuid]/summary` cold launches (e.g., from push notifications). Must verify Capacitor actually serves `index.html` for unmatched paths.

2. **`/session/_/summary` is the only pre-rendered session path.** If a user somehow cold-launches to this exact URL, they'll see a "session not found" state (the component gracefully handles missing sessions). All real session UUIDs rely on the `errorPath` fallback routing through `index.html` then the Next.js client router.

3. **Supabase auth email redirect URLs** — Supabase is configured to redirect to `https://gymhub-inky.vercel.app/auth/callback`. This is NOT changed by this migration. Email confirmation links (signup, password reset) still point to Vercel. This is intentional — changing Supabase redirect URLs was explicitly out of scope. The auth callback page works fine for in-app navigation (login form → Supabase → local webview). For email-based auth flows, the user will be redirected to Safari, not the app. This is a known limitation.

4. **`@vercel/analytics`** — `app/layout.tsx` includes Vercel Analytics. In a static Capacitor bundle, these outbound requests may be blocked by App Transport Security or fail silently. The app will not crash, but analytics data from the iOS app will not be reported. Low priority — does not affect user experience.

5. **Offline behavior** — The static bundle means all UI assets load from disk. Supabase data calls (sessions, profile) still require network. Verify that Supabase client gracefully handles offline scenarios (it should, as it uses browser-native fetch with localStorage fallback patterns).

6. **`trailingSlash: true` and existing deep links** — Any external links or bookmarks to the Vercel deployment use non-trailing-slash URLs. These are unaffected (Vercel continues to serve the web version). Within the Capacitor app, all navigation uses `router.push()` so trailing slash routing is handled transparently.

## Go / No-Go recommendation

**Go — with one caveat to verify.**

The build is clean, all 35 pages compile, TypeScript passes, and `cap sync` completed successfully. The iOS bundle now contains the full Next.js static app. The architectural changes are minimal and correct.

The single item requiring device verification before committing to merge: **confirm that `server.errorPath: "index.html"` actually activates SPA fallback routing in Capacitor's local server on iOS.** This is the mechanism that makes dynamic route cold launches work. If it does not work as expected, deep-linked session/edit/exercise URLs will 404 on cold launch (in-app navigation still works fine).

If the `errorPath` approach fails, the alternative is to create a `next.config.ts` post-build step that copies `out/index.html` to `out/404.html` (Next.js may already generate `out/404.html`), which Capacitor might use as the error page. This was verified to exist in the `out/` directory in this build.

Everything else is low-risk. The app was already 100% client-side — no server components, no middleware, no API routes — so the static export is a configuration change, not an architectural one.
